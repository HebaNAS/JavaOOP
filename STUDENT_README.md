# Java OOP Quest — Student Guide

Welcome to **Java OOP Quest**, the companion tool for **F27SA Software Development 1**.

Learn Java by writing real code that compiles and runs — no installation needed. Just open the link in your browser and start coding.

---

## Getting Started

1. Open the application in any web browser (Chrome, Firefox, Safari, Edge)
2. You will see the **home page** with all available units
3. Click a unit to begin its challenges
4. Write Java code in the editor on the left
5. Press **Run** (or **Ctrl+Enter**) to compile and test your code
6. Read the feedback — fix any errors and try again
7. Complete a challenge to earn XP and unlock the next one

No Java installation is required on your computer. Your code is compiled on the server using the real Java compiler (`javac`).

The course is divided into units, each with a theme and a set of coding challenges:

## Challenge Types

Most challenges ask you to **write code** that produces specific output. But there are also:

- **Predict the Output** — You are shown code and must type what you think it will print. Tests your code-reading skills.
- **Find the Bug** — You are given broken code marked with a bug icon. Find and fix the error so it produces the correct output.

---

## The Battle Arena (Units 7-9)

The arena is a special 3D environment where your Java classes come to life:

- Define a `Warrior` class and use `new Warrior("Aldric", 100, 25)` to summon a character
- Add methods like `attack()`, `defend()`, `castSpell()`, `heal()` to give your characters abilities
- Use inheritance to create `Mage`, `Archer`, and `Healer` subclasses
- Control characters with your keyboard (WASD to move, Space to attack, Q to defend)
- Fight enemies, build parties with `ArrayList`, and defeat the final boss

The arena validates your code structure **and** compiles it with the real Java compiler. You will see both game feedback and compiler messages in the console.

## XP, Levels, and Achievements

- Every challenge you complete earns **XP**
- Harder challenges earn more XP
- Your total XP and progress are shown on the home page
- **Achievements** are badges you earn for milestones:
  - Complete your first challenge
  - Clear an entire unit
  - Fix a bug or predict output correctly
  - Maintain a coding streak (3 or 7 days)
  - Reach XP milestones (500, 2000)
- **Streaks** — code every day to build a streak counter

---

## Hints

Each challenge offers up to 3 hints, progressively more specific:

1. **Conceptual** — explains the idea
2. **Structural** — tells you the syntax pattern
3. **Specific** — nearly gives the answer

Using hints does not reduce your XP, but try solving challenges without them first.

---

## Saving Your Work

- Your code is **auto-saved as a draft** as you type
- When you complete a challenge, your solution is saved
- All progress is stored in your browser (localStorage)
- If you clear your browser data, your progress will be lost

## Pro Tips

- Using IDEs (IntelliJ, VS Code) for code completion and error highlighting

- Version control with Git (commit, push, branch)

## Troubleshooting

| Problem | Solution |
|---------|----------|
| "Cannot reach compilation server" | The backend is not running. Ask your instructor. |
| Code compiles but challenge says wrong output | Check for extra spaces, missing newlines, or wrong capitalisation. Output must match exactly. |
| My progress disappeared | Progress is stored in your browser. Clearing browser data erases it. |
| The 3D arena is slow | Try closing other tabs. The arena uses WebGL which needs GPU resources. |
| I'm stuck on a challenge | Use all 3 hints. Try a simpler approach. Ask a classmate or your instructor. |
